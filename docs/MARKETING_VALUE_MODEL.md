## 0. 目的
- 「機能」ではなく「体験」で売るための背骨を固定する
- "驚かせポイント"を一貫させ、価格とバイラル導線を正当化する
- 技術実装（docs）とマーケ主張がズレないようにする

## 1. 驚かせポイント（体験価値）— 3本に固定
日程調整は機能が似る。勝つのは「やった瞬間に脳が気持ちよくなる体験」。

### 驚き①：考えなくても予定が"勝手に決まる"（決定ストレスゼロ）
ユーザーの本音は「日程調整したい」ではない。
- 何往復もしたくない
- 失礼なやり取りをしたくない
- 決めるのがしんどい

tomonowaは「投げたら終わる」を作る。
- 候補生成 → 送信 → 回答 → 集計 → 主催者確定 → Meet生成 → カレンダー反映

※重要: **無限再調整禁止（最大2回）**が "終わる体験" を保証する（SCHEDULING_RULES準拠）。

### 驚き②：難しい条件（AND/CORE/MAX）が当たり前にできる（なのに簡単）
ユーザーは条件をUIで選びたくない。自然言語で言いたい。
- 全員必須（AND）
- コア必須（CORE）
- 最大参加（MAX）

チャットUXなら自然に言える（UX_CHAT_SPEC準拠）。
フォームで条件選択させるのは禁止（SCHEDULING_RULES準拠）。

### 驚き③：人間関係の摩擦が消える（気まずさゼロ）
- 催促しなくていい
- 断りが気まずくならない
- リスケがしんどくない
- 募集・集計が勝手にまとまる

「通知をうざくしない」ための **チャネル分離（Chat/Bell/Email）**が驚きの一部（NOTIFICATION_CHANNEL_RULES準拠）。

## 2. 顧客が買う"無意識の欲求"に翻訳（欲求/恐怖/渇望）

### 2.1 欲求（こうなりたい）
- ちゃんとしてる人に見られたい
- 気が利く主催者でいたい
- 忙しいのに回せてる自分でいたい
- "できる人"の自己像が欲しい

=> 生産性ではなく **「できる人の自己像」**を売る。

### 2.2 恐怖（こうなりたくない）
- 調整ミスで信用を落としたくない
- 催促で嫌われたくない
- 抜け漏れで機会を逃したくない（商談/面接/採用/運営）

=> 恐怖が強い層ほど単価が上がる（営業/採用/運営/講師/主催）。

### 2.3 渇望（人生テーマ）
- 自由：予定に縛られたくない
- 安心：抜け漏れゼロでいたい
- 承認：信頼されたい
- 役割：人をつなぐハブでいたい

=> 渇望に刺さると **習慣ツール化**し、解約しにくい。

## 3. 価格の正当化（値付けの芯）
「機能数」ではなく「削減するストレス」で価格を作る。

### 3.1 3つのコスト削減
1) 時間コスト：往復の削減  
2) 感情コスト：気まずさ・催促・断りの削減  
3) 信用コスト：抜け漏れ・ミス・失注の削減  

=> 「月◯円で"信用の保険"を買う」構造になり、高単価でも通る。

### 3.2 価格帯の考え方（概念）
- 外部リンク調整（external中心）：低〜中（量より導線）
- coworker/チーム運用（Room/Grid）：中〜高（摩擦削減価値が増大）
- イベント/募集/集計（MAX）：中〜高（主催者の信用コストを直撃）

---

## 4. バイラル設計（仕様と矛盾させない）
最も伸びるのは「外部招待（未登録者）」を入口にする導線。

### 4.1 外部招待の導線（最小）
1) 主催者が external に送る（メール + 調整リンク）  
2) 相手が /i/:token で回答（登録不要）  
3) 確定するとMeetが生成される（価値体験）  
4) "次回からアプリ内で受け取れる" → 登録導線  
5) 登録後は Bell（軽通知）へ昇格  
6) coworker化したら Chat（本丸体験）へ昇格  

※通知チャネルは NOTIFICATION_CHANNEL_RULES の関係性遷移ルールに従う。

### 4.2 "うざい"回避は仕様で担保する
- externalの無限再調整は禁止  
- 調整ラウンドは最大2回（SCHEDULING_RULES準拠）  
=> これがあるからバイラルが嫌われない。

---

## 5. 関係性別の価値訴求（メッセージングの芯）
関係性で「刺さる価値」が変わる。

### family
- "勝手に整う"（結果だけ通知）
- 予定ブロック・旅行・家庭行事が一瞬

### coworker（Room/Grid）
- "摩擦ゼロで決まる"（裏で調整、結果だけ）
- チームの信用を守る

### external
- "相手に負担をかけない"リンク調整
- 必要以上に通知しない（最大2回）
